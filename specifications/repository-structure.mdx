## Module rules

1. Domain packages depend only on the shared kernel.
2. Application packages depend on domain packages and port interfaces.
3. Runtime kernel depends on execution-domain types and outbound interfaces.
4. Adapters implement ports and translate external DTOs at the boundary.
5. Apps compose modules and own process startup/configuration.
6. Tests may depend inward; production core never depends on tests or adapters.
7. No bounded context reads another contextвЂ™s database tables directly.
8. Integration crosses published API/event contracts.

## Initial physical deployment

Do not deploy every directory as a service. Start with a modular control-plane application, runtime API/workers, sandbox workers, evaluation workers, and managed infrastructure. Extract services only for scale, security, availability, residency, team ownership, or independent commercial lifecycle.

# Reference interfaces

> **Status: Normative.** This page is the canonical port registry. Examples use TypeScript-like pseudocode.

## Shared contexts and errors

```typescript
type Result<T, E = PortError> =
  | { ok: true; value: T }
  | { ok: false; error: E };

type SideEffectClass =
  | "pure"
  | "read_only"
  | "idempotent_write"
  | "reversible_write"
  | "irreversible_write";

interface ExecutionContext {
  tenant: TenantScope;
  actor: ActorIdentity;
  runId?: RunId;
  activityRunId?: ActivityRunId;
  correlationId: string;
  deadline: Instant;
  dataClassification: DataClassification;
  residencyRegion: Region;
  capabilityGrant: CapabilityGrant;
}

interface InvocationContext extends ExecutionContext {
  effectId: EffectId;
  invocationAttemptId?: InvocationAttemptId;
}

interface PortError {
  category: ErrorCategory;
  code: string;
  message: string;
  retryable: boolean;
  safeToRetry: boolean;
  effectMayHaveOccurred: boolean;
  retryAfter?: Duration;
  externalReference?: string;
}
```

## Execution kernel

```typescript
interface RunMachine {
  evolve(state: RunState, event: RunEvent): RunState;
  decide(
    state: RunState,
    definition: WorkflowDefinition,
    deployment: DeploymentSnapshot
  ): readonly RuntimeIntent[];
}

type RuntimeIntent =
  | { type: "start_activity"; activity: ActivityId; input: InputSnapshotRef }
  | { type: "schedule_activity_retry"; activityRunId: ActivityRunId; at: Instant }
  | { type: "request_approval"; request: ApprovalRequest }
  | { type: "wait_for_event"; wait: ExternalEventWait }
  | { type: "start_child_run"; delegation: ChildRunRequest }
  | { type: "write_checkpoint" }
  | { type: "complete"; result: ExecutionResultDraft }
  | { type: "fail"; failure: RunFailure };
```

## Contract-test requirements

Each adapter must prove error classification, timeout semantics, cancellation, idempotency, tenant scoping, content redaction, usage accuracy, streaming completion, and capability negotiation. Mutating tool adapters additionally prove reconciliation after connection loss. State/journal/workflow/evidence adapters prove tenant isolation, expected-version behavior, duplicate delivery handling, and required durability.

# Reference interfaces

The examples use TypeScript-like pseudocode. Production code may use TypeScript, Python, Java, Go, C#, or another typed language while preserving semantics.

## Shared types

```typescript
type Result<T, E = PortError> =
  | { ok: true; value: T }
  | { ok: false; error: E };

type SideEffectClass =
  | "pure"
  | "read_only"
  | "idempotent_write"
  | "reversible_write"
  | "irreversible_write";

interface InvocationContext {
  tenant: TenantScope;
  actor: ActorIdentity;
  runId?: RunId;
  activityRunId?: ActivityRunId;
  attemptId?: AttemptId;
  effectId?: EffectId;
  correlationId: string;
  deadline: Instant;
  dataClassification: DataClassification;
  residencyRegion: Region;
  capabilityGrant: CapabilityGrant;
}
```

## Execution kernel

```typescript
interface RunMachine {
  evolve(state: RunState, event: RunEvent): RunState;
  decide(
    state: RunState,
    definition: WorkflowDefinition,
    deployment: DeploymentSnapshot
  ): readonly RuntimeIntent[];
}

type RuntimeIntent =
  | { type: "start_activity"; activity: ActivityId; input: InputSnapshotRef }
  | { type: "schedule_retry"; activityRunId: ActivityRunId; at: Instant }
  | { type: "request_approval"; request: ApprovalRequest }
  | { type: "wait_for_event"; wait: ExternalEventWait }
  | { type: "start_child_run"; delegation: ChildRunRequest }
  | { type: "write_checkpoint" }
  | { type: "complete"; result: ExecutionResultDraft }
  | { type: "fail"; failure: RunFailure };
```

## Model port

```typescript
interface ModelPort {
  capabilities(
    ctx: InvocationContext,
    model: ModelRef
  ): Promise<Result<ModelCapabilities>>;

  invoke(
    ctx: InvocationContext,
    request: ModelRequest
  ): AsyncIterable<ModelEvent>;

  cancel(
    ctx: InvocationContext,
    invocation: ModelInvocationId
  ): Promise<Result<void>>;
}

type ModelEvent =
  | { type: "started"; invocationId: ModelInvocationId }
  | { type: "content_delta"; text: string }
  | { type: "tool_call_delta"; call: PartialToolCall }
  | { type: "usage_delta"; usage: PartialUsage }
  | { type: "completed"; result: ModelResult; usage: Usage }
  | { type: "failed"; error: PortError };
```

Provider-specific capabilities may appear through typed extensions included in the deployment snapshot. Do not pass raw provider request objects through the core.

## Tool port

```typescript
interface ToolPort {
  describe(ctx: InvocationContext, tool: ToolRef): Promise<Result<ToolDescriptor>>;
  invoke(ctx: InvocationContext, request: ToolRequest): Promise<Result<ToolResponse>>;
  status(ctx: InvocationContext, operation: ExternalOperationRef): Promise<Result<OperationStatus>>;
  cancel(ctx: InvocationContext, operation: ExternalOperationRef): Promise<Result<void>>;
}

interface ToolRequest {
  tool: ToolRef;
  arguments: unknown;
  argumentsDigest: ContentDigest;
  sideEffectClass: SideEffectClass;
  idempotencyKey?: string;
  credentialRef?: SecretRef;
}

type ToolResponse =
  | { status: "completed"; result: ToolResult }
  | { status: "accepted"; operation: ExternalOperationRef }
  | { status: "requires_polling"; operation: ExternalOperationRef };
```

## Policy port

```typescript
interface PolicyPort {
  evaluate(
    ctx: InvocationContext,
    request: PolicyRequest
  ): Promise<Result<PolicyDecision>>;
}

type PolicyDecision =
  | { decision: "allow"; policyVersions: PolicyVersionRef[]; obligations: Obligation[] }
  | { decision: "deny"; policyVersions: PolicyVersionRef[]; reasons: PolicyReason[] }
  | {
      decision: "require_approval";
      policyVersions: PolicyVersionRef[];
      requirement: ApprovalRequirement;
      obligations: Obligation[];
    };
```

## Contract test requirements

Each adapter must prove error classification, timeout semantics, cancellation, idempotency, tenant scoping, content redaction, usage accuracy, streaming completion, and capability negotiation. Mutating tool adapters additionally prove reconciliation after connection loss.

# Canonical conventions

> **Status: Normative.** Schemas, APIs, manifests, examples, telemetry mappings, and adapters follow this profile.

## Naming grammar

| Scope | Rule | Examples |
|---|---|---|
| Stable catalog identity | Plain singular noun | `Agent`, `Workflow`, `Tool`, `Prompt`, `Policy` |
| Immutable behavior/configuration | `Version` suffix | `AgentVersion`, `WorkflowVersion`, `ToolVersion` |
| Independently durable execution | `Run` suffix | `WorkflowRun`, `AgentRun`, `ExecutionPlanRun` |
| Subordinate retry | Qualified `Attempt` | `ActivityAttempt`; never generic `AttemptRun` |
| Intentional sequential repetition | Plain noun | `Iteration` |
| Parallel path/candidate | Qualified noun | `ExecutionBranch` |
| Logical external/nondeterministic operation | Plain noun | `Effect` |
| Concrete adapter/provider call | Plain noun | `Invocation` |
| Temporary worker ownership | `Lease` suffix | `WorkerLease` |
| Experiment replicate | Qualified domain noun | `ExperimentTrial` |
| Immutable execution input | `Snapshot` suffix | `InputSnapshot`, `ContextSnapshot`, `DeploymentSnapshot` |
| Committed output | `Result` suffix | `ActivityResult`, `EvaluationResult` |
| Immutable content | `Artifact` | `Artifact`, `EvidenceArtifact` |

The normative core does not define unqualified `Step`, `Round`, `AttemptRun`, `TrialRun`, `SubAgent`, `SubWorkflow`, or `ExecutionEpisode`.

Qualified names are allowed where their semantics are explicit: `PlanStep`, `UIProgressStep`, `DeliberationRound`.

## Resource references

Human-readable immutable references use:

```text
<name>@<semantic-version>
```

Example: `purchase-order.create@4.2.1`.

Production deployments resolve every reference to an exact content digest. Versions communicate compatibility intent; digests identify exact content.

## Identifiers

Identifiers are opaque strings with a type prefix where useful:

```text
run_01K2A
activity_run_12
activity_attempt_2
effect_01K2B
invocation_01K2I
worker_lease_01K2L
```

Clients MUST NOT infer tenant, time, order, or authority from an identifier.

## Money and quantities

Money is a decimal string plus ISO 4217 currency:

```json
{ "amount": "8.00", "currency": "USD" }
```

Binary floating-point serialization is prohibited for money.

## Time and duration

- Instants use RFC 3339 UTC timestamps.
- Durations use ISO 8601 durations.
- Deadlines are absolute instants.
- Timeouts are durations.

## Event naming

Canonical serialized event types use dotted lowercase:

```text
run.started
activity.completed
effect.planned
invocation.started
invocation.completed
approval.requested
```

Language classes may be PascalCase if an explicit adapter mapping exists.

## Version and sequence fields

| Field | Meaning |
|---|---|
| `schemaVersion` | Serialized contract version |
| `stateVersion` | Optimistic-concurrency version of current state |
| `aggregateVersion` | Version of the emitting aggregate |
| `runSequence` | Monotonic event order within one run |
| `attemptOrdinal` | Order of qualified activity attempts |
| `invocationOrdinal` | Order of invocations for one effect |
| `iterationOrdinal` | Order of intentional iterations |
| `trialOrdinal` | Order of experiment trials within a variant |

These fields are not interchangeable.

## Properties and enums

Serialized property names use `camelCase`. Enum values use `snake_case` unless an external protocol requires another representation.

## Context and tenancy

Every protected state, artifact, memory, event, audit, usage, and adapter operation carries trusted tenant/workspace context derived from authenticated identity and routing.

## Payloads

Large or sensitive content is stored by immutable reference and digest. Canonical events contain metadata and protected payload references rather than duplicating full content.

## Optional values

Omitted means not supplied or not applicable. `null` is used only when the schema assigns it a distinct meaning.

## Compatibility and migration

- Published versions and historical events are immutable.
- Compatible schema changes use new versions and readers/upcasters.
- Renames require an alias/deprecation map for a bounded migration.
- Production manifests contain no unresolved `latest` aliases or version ranges.
- Examples that share an identifier describe the same scenario.

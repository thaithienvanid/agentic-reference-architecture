## Event categories

- Domain events carry business meaning.
- Execution events describe run lifecycle.
- Integration events cross bounded contexts through the outbox.
- Audit events carry authoritative security/governance evidence.
- Telemetry signals support operations and may be sampled.

Do not force them into one retention or access model.

## State transaction

```text
BEGIN
  verify expected stateVersion and runSequence
  append canonical events
  update run-state projection
  update effect and budget ledgers
  write integration outbox
COMMIT
```

## Schema evolution

Events carry explicit semantic versions. Use readers/upcasters for compatible evolution. Never rewrite old events to pretend that historical executions used newer semantics.

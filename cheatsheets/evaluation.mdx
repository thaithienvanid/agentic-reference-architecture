# Evaluation-driven development cheatsheet

> **Write the evaluation contract before changing behavior.**

## Loop

```text
requirement or incident
-> add deterministic and evaluation cases
-> implement smallest change
-> run unit/contract/replay tests
-> run semantic and trajectory evals
-> run environment and security tasks
-> enforce promotion gates
-> canary or shadow
-> curate production failures
```

## Tool split

| Tool | Best use |
|---|---|
| Unit/property tests | Domain invariants and state transitions |
| Port contract suite | Provider/adapter behavior |
| DeepEval | Semantic, plan, tool, argument, task, and trajectory quality |
| Harbor | Sandboxed final-state and operational verification |
| Human review | Nuanced domain judgment and calibration |
| Online evaluation | Production drift and outcomes |

```text
DeepEval asks: Did the agent behave well?
Harbor asks: Did the environment end correctly?
The platform asks: Was execution authorized, durable, isolated, and auditable?
```

## Hard gates

```text
zero cross-tenant access
zero approval bypass
zero forbidden tool or egress
zero duplicate irreversible effect
no missing mandatory audit evidence
hard budget enforced before dispatch
```

Never average a hard-gate failure into a high quality score.

## DeepEval projection

```text
Run events and artifacts
-> evaluation trace projector
-> agent / model / tool / context spans
-> component and end-to-end metrics
-> platform EvaluationResult
```

## Harbor task

```text
instruction
+ pinned container/environment
+ fixtures and failure injection
+ deterministic verifier/tests
+ repeated trials
-> final-state evidence
```

## CI tiers

| Stage | Evaluation |
|---|---|
| Developer | Fast unit, mocks, small golden slice |
| Pull request | Contracts, replay, representative DeepEval, Harbor smoke |
| Nightly | Full datasets, repeated trials, adversarial/fault injection |
| Release | Production configuration and critical cases |
| Canary | Real or shadow traffic with rollback triggers |
| Production | Online sampling and incident curation |

## Reproducibility

Pin the subject versions, deployment digest, dataset, evaluator, judge model/profile, environment digest, trial count, random controls when supported, and all evidence references.

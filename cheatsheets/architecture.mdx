# Agentic Reference Architecture cheatsheet

> **Models propose. Deterministic code and policy authorize. The runtime executes, persists, and records.**

## Canonical resource model

```text
Agent -> AgentVersion -> AgentRun
Workflow -> WorkflowVersion -> WorkflowRun
ExecutionPlan -> ExecutionPlanVersion -> ExecutionPlanRun
Prompt -> PromptVersion -> CompiledPromptSnapshot
Tool -> ToolVersion -> Effect -> Invocation
Policy -> PolicyVersion -> PolicyDecision
EvaluationSuite -> EvaluationSuiteVersion -> EvaluationRun
```

Stable resource identity is not executable. Published versions are immutable. Runs pin an exact deployment snapshot.

## Canonical execution model

```text
ExecutionPlanRun
└── WorkflowRun
    └── ActivityRun
        ├── ActivityAttempt    whole-activity restart
        ├── ExecutionBranch   parallel candidate/path
        ├── Iteration         sequential cycle with updated state
        └── Effect            logical external/nondeterministic operation
            └── Invocation    concrete adapter/provider call

WorkerLease                  temporary worker ownership/fencing
```

Trials are not core runtime children:

```text
ExperimentRun -> ExperimentVariant -> ExperimentTrial -> WorkflowRunRef
```

## Naming rules

| Meaning | Canonical name |
|---|---|
| Semantic work in a workflow version | `Activity` |
| Logical execution of that work | `ActivityRun` |
| Whole activity restarted after failure | `ActivityAttempt` |
| Worker resumes the same durable work | New `WorkerLease` |
| Intentional cycle with updated state | `Iteration` |
| Parallel candidate or model variant | `ExecutionBranch` or domain `ExperimentVariant` |
| Independent experiment replicate | `ExperimentTrial` |
| Logical model/tool/memory/human operation | `Effect` |
| One concrete external call | `Invocation` |
| Proposed plan item | `PlanStep` |
| Domain discussion cycle | `DeliberationRound` mapped to `Iteration` |

Do not create unqualified `Step`, `Round`, `AttemptRun`, `TrialRun`, `SubAgent`, or `SubWorkflow` entities.

## Kernel and runtime

| Component | Owns |
|---|---|
| Domain | Business meaning and invariants |
| Application | Commands, queries, and use cases |
| Execution Kernel | Pure legal state evolution and execution intents |
| Runtime Service | Admission, coordination, scheduling, cancellation, recovery |
| Scheduler | Fairness, capacity, priority, placement |
| Runtime Worker | Bounded assigned work |
| WorkerLease | Temporary ownership and fencing |
| DurableExecutionPort | Timers, signals, suspension, backend recovery |
| Platform Run Journal | Canonical execution facts |
| Effect Ledger | Effect identity, invocation history, reconciliation |
| Gateways | Models, tools, memory, artifacts, humans, sandboxes |

## Effect flow

```text
proposal
-> validate
-> authorize
-> reserve budget
-> approve when required
-> persist effect
-> invoke
-> reconcile outcome and usage
-> advance state
```

A timeout does not prove a mutation failed.

## State boundaries

```text
Domain state       business truth
Workflow state     next legal transition
Input snapshot     exact activity input
Activity result    typed committed output
Context snapshot   exact model context
Memory record      reusable governed information
Artifact           immutable content
Checkpoint         resume optimization
Run journal        canonical execution facts
Trace              operational diagnosis
Audit              authoritative control evidence
Usage              cost attribution
```

## Child rule

Inline execution shares the parent lifecycle and creates no independent run. Child execution creates a separate `WorkflowRun` or `AgentRun` with explicit task, capabilities, budget, deadline, result contract, and cancellation policy.

## Production default

```text
modular control plane
+ execution kernel library
+ runtime service and workers
+ durable execution adapter
+ relational state/journal/effect ledger
+ immutable artifact storage
+ model/tool/policy/secret gateways
+ evaluation workers
+ OpenTelemetry-compatible observability
```

## Profiles

- **ARA Core:** versions, workflows, activities, effects, ports, state/evidence separation.
- **ARA Durable:** journal, leases, idempotency, reconciliation, resume, replay.
- **ARA Enterprise:** tenancy, identity, audit, quotas, SLOs, cells.
- **ARA Marketplace:** signed packages, permissions, sandbox, revocation.
- **ARA Regulated:** human oversight, data rights, retention, tamper evidence.

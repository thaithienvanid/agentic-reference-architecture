# Gates, metrics, and reproducibility

> **Status: Normative reference for the canonical hard-gate registry; informative for metric examples.**

## Promotion rule

```text
Deterministic invariants pass
AND semantic and trajectory thresholds pass
AND environment and security tasks pass
AND cost and latency remain within limits
AND every applicable canonical hard gate passes
AND no critical regression exists
```

Do not collapse all dimensions into one weighted average. A package with quality 0.96 and one approval bypass fails.

## Canonical hard-gate registry

The registry is `enterprise-agent-safety@1.0.0`:

```text
no_cross_tenant_access
no_unauthorized_tool_execution
no_approval_bypass
approval_action_digest_matches
no_duplicate_irreversible_effect
no_forbidden_data_egress
hard_budget_enforced
mandatory_audit_complete
```

Domain suites may add stricter gates but must not remove these when the corresponding capability or risk is present. A non-applicable gate is explicitly `not_applicable` with evidence; omission is not a pass.

## Metric ownership

| Subject | Metrics |
|---|---|
| Agent version | Task success, trajectory, policy compliance, cost, latency, escalation |
| Workflow version | Branch frequency, failure points, duration, completion, retry burden |
| Dataset version | Coverage, freshness, difficulty, slice distribution, label agreement |
| Workflow/agent run | Terminal state, quality, policy, cost, active/suspended time, recovery |
| Activity run | Branches, iterations, effects, latency, usage, progress contribution |
| Experiment trial | Outcome distribution, variance, cost, latency, critical-slice failures |
| Effect | Authorization, outcome, ambiguity, idempotency/reconciliation |
| Invocation | Adapter latency, retry/poll/reconciliation role, provider usage/error |
| Model route | Quality, schema adherence, refusal, tool accuracy, cost, latency |
| Tool version | Success, timeout, ambiguous outcome, idempotency, authorization denial |
| Tenant | Spend, quota, SLO, incidents, quality distribution |
| Package release | Installs, rollback, findings, resource cost, evaluation score |

## Reproducing a failure

Capture subject versions, deployment snapshot, baseline and experiment-trial metadata where applicable, normalized model requests/results or protected digests, selected context, tool arguments/results, effects and invocations, approvals, budget state, worker leases, adapter/environment versions, state-transition events, and artifacts.

Exact token regeneration is not generally guaranteed. Evidence replay can reproduce deterministic state evolution using recorded effect outcomes. Counterfactual reruns create new runs and identify which effects were reinvoked.

## Online evaluation

Evaluate every policy/security failure, sample successful runs by risk, use shadow execution without applying mutating effects, monitor slice-level drift, and curate reviewed incidents into versioned datasets.

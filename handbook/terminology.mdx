# Terminology and mental model

> **Status: Informative.** Canonical execution semantics are defined in the [ARA specification](/rfc/index).

## Product concepts

| Concept | Boundary |
|---|---|
| **LLM-powered application** | A model contributes to an output while application code owns control flow. |
| **Assistant** | A user-facing help relationship; it may or may not be agentic. |
| **Agent** | Stable catalog identity for an agent capability. |
| **AgentVersion** | Immutable executable behavior: instructions, model requirements, capabilities, policies, budgets, and evaluation. |
| **AgentRun** | Independent durable execution of an `AgentVersion`. |
| **Agentic workflow** | A workflow where a model proposes at least one plan, branch, delegation, or tool choice. |
| **Multi-agent system** | Independently governed agent runs with explicit contracts and authority. |
| **Agentic service** | Callable capability exposed through APIs/events. |
| **Agentic application** | Domain product with use cases, state, UX, policy, and workflows. |
| **Agentic platform** | Shared control, execution, governance, evaluation, tenancy, and operations. |

## Execution vocabulary

| Term | Meaning |
|---|---|
| `WorkflowVersion` | Immutable executable process specification |
| `WorkflowRun` | One durable execution |
| `Activity` | Semantic unit inside a workflow version |
| `ActivityRun` | One logical execution of an activity |
| `ActivityAttempt` | Whole-activity restart from the declared retry boundary |
| `ExecutionBranch` | Independent parallel path/candidate |
| `Iteration` | Intentional sequential cycle using updated state/evidence |
| `Effect` | Logical external or nondeterministic operation |
| `Invocation` | One concrete adapter/provider call for an effect |
| `WorkerLease` | Temporary worker ownership and fencing |
| `ExperimentTrial` | Independent experimental repetition from a frozen baseline |

`Run` is reserved for independently durable scopes.

## Inline and child

- **Inline** work shares the parent lifecycle and has no independent run identity.
- **Child** work creates a separate run with its own state, result, budget, deadline, failure, and capability grant.

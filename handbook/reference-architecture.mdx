# Reference architecture overview

> **Status: Informative overview.** Normative requirements live under [ARA specification](/rfc/index).

```mermaid
flowchart TB
    User["Users, applications, developers, operators"]

    subgraph Product["Agentic application or service"]
      App["Application use cases"]
      Domain["Domain bounded contexts"]
      Plans["Execution plans"]
    end

    subgraph Control["Control plane"]
      Resources["Resources and immutable versions"]
      Compiler["Deployment compiler"]
      Tenant["Tenant and governance configuration"]
    end

    subgraph Execution["Execution plane"]
      API["Run API and streams"]
      Runtime["Runtime Service and Scheduler"]
      Kernel["Execution Kernel"]
      Journal["State, journal, effect ledger"]
      Gate["Policy, budget, approval"]
    end

    subgraph Ports["Capability and infrastructure ports"]
      Durable["DurableExecutionPort"]
      Model["ModelPort"]
      Tool["ToolPort"]
      Memory["MemoryPort"]
      Sandbox["SandboxPort"]
    end

    subgraph Evidence["Evaluation and evidence"]
      Eval["Evaluation"]
      OTel["Observability"]
      Audit["Audit and lineage"]
      Meter["Usage and billing"]
    end

    User --> App
    App --> Domain
    App --> Plans
    Plans --> API
    Resources --> Compiler
    Tenant --> Compiler
    Compiler -. signed snapshot .-> Runtime
    API --> Runtime --> Kernel
    Kernel <--> Journal
    Kernel --> Gate --> Ports
    Journal --> Eval
    Runtime --> OTel
    Journal --> Audit
    Journal --> Meter
```

Source dependencies point inward. Domain and application code do not import model SDKs, durable backends, databases, brokers, cloud APIs, or observability vendors.

Read [Resource and version model](/rfc/resource-model), [Execution model](/rfc/execution-model), [Runtime and kernel](/rfc/runtime), and [Platform architecture](/rfc/platform).

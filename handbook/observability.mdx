# Observability, audit, and lineage

> Observability explains operations. It does not replace run state, audit, evaluation, or billing.

## Evidence separation

| Record | Answers |
|---|---|
| Run journal | What authoritatively happened? |
| Trace | What causal path and latency occurred? |
| Metric | What aggregate health and trend exists? |
| Structured log | What diagnostic event occurred? |
| Evaluation | Was behavior good, safe, and correct? |
| Audit | Who authorized and executed what? |
| Usage ledger | What consumption is attributable to whom? |

## Trace hierarchy

```text
execution-plan.run
  workflow.run
    activity.run
      context.compile
        memory.search
        retrieval.query
        prompt.compile
      policy.evaluate
      model.invoke
      tool.invoke
      artifact.write
      child workflow.run
    approval.wait
    evaluation.online
```

One activity may produce several spans. Do not force a one-node/one-span mapping.

Run IDs belong in traces and logs, not high-cardinality metric labels.

## Agentic golden signals

Traditional latency, traffic, errors, and saturation are extended with quality, cost, progress, and safety.

Progress signals include new evidence, contradictions resolved, required fields completed, score improvement, repeated action count, and cost delta. Alert when iterations consume cost without state or quality progress.

## Metrics

- Run starts, terminal states, active/suspended duration, recovery latency.
- Model latency, tokens, cost, rate limits, invalid structured output.
- Tool success, timeout, ambiguous outcome, reconciliation, policy denial.
- Agent iterations, delegation, no-progress, loop detection, escalation.
- Context tokens, compaction, memory policy rejection.
- Approval count, denial, expiry, wait duration, revalidation failure.
- Evaluation scores and regressions.
- Tenant queue fairness and sandbox saturation.

## Content handling

Do not collect raw secrets, authorization headers, private hidden reasoning, or full sensitive prompts by default. Record versions, digests, source references, token counts, and protected payload references. Apply redaction before telemetry export.

## Sampling

Keep 100% of failed runs, security events, policy denials, ambiguous effects, budget stops, sandbox findings, and evaluation regressions. Tail-sample successful low-risk runs and high-volume read-only effects.

## Cross-run causality

Use a separate trace per workflow run, a shared execution-plan/experiment ID, span links, and explicit artifact/decision lineage. Avoid week-long monolithic traces.

## Observability tests

Contract tests should prove every run creates a root span, important effects correlate with journal event IDs, usage is recorded, sensitive fields are absent, failed traces survive sampling, cross-run links exist, and metric labels remain bounded.

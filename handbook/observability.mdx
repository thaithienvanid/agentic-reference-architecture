# Observability, audit, and lineage

> Observability explains operations. It does not replace run state, audit, evaluation, or billing.

## Evidence separation

| Record | Answers |
|---|---|
| Run journal | What authoritatively happened? |
| Trace | What causal path and latency occurred? |
| Metric | What aggregate health and trend exists? |
| Structured log | What diagnostic event occurred? |
| Evaluation | Was behavior good, safe, and correct? |
| Audit | Who authorized and executed what? |
| Usage ledger | What consumption is attributable to whom? |

## Trace hierarchy

```text
execution-plan.run
  workflow.run
    activity.run
      branch / trial / iteration (when present)
      effect
        invocation.attempt
      context.compile
      policy.evaluate
      approval.wait
    evaluation.online
```

One activity may produce several effects and spans. Do not force a one-node/one-span mapping.

## Stable attributes

```text
tenant.id_hash, workspace.id_hash, region
execution_plan_run.id, workflow.run.id, activity.run.id
branch.id, trial.id, iteration.id
effect.id, invocation_attempt.id
workflow/agent/prompt/tool/policy versions, deployment.digest
model.route/provider_identifier
side_effect.class, data.classification
usage.input_tokens/output_tokens/cost
retry.count, completion.reason, fallback.route, cache.status
```

Run and invocation IDs belong in traces/logs, not high-cardinality metric labels.

## Agentic golden signals

Traditional latency, traffic, errors, and saturation are extended with quality, cost, progress, and safety. Progress includes new evidence, contradictions resolved, fields completed, score improvement, repeated-action count, and cost delta.

## Metrics

- Run starts, terminal states, active/suspended duration, recovery latency.
- Model latency, tokens, cost, rate limits, invalid output, cache use, route/fallback.
- Tool success, timeout, ambiguous outcome, reconciliation, policy denial.
- Branch/trial/iteration count, variance, marginal quality, and abandoned work.
- Agent delegation, no-progress, loop detection, escalation.
- Context tokens, compaction, retrieval and memory-policy rejection.
- Approval count, denial, expiry, wait duration, revalidation failure.
- Evaluation scores/regressions, tenant fairness, sandbox saturation.
- Cost per successful evaluated outcome.

## Content handling

Do not collect raw secrets, authorization headers, private hidden reasoning, or full sensitive prompts by default. Record versions, digests, source references, token counts, and protected payload references. Apply redaction before export.

## Sampling

Keep all failed runs, security events, policy denials, ambiguous effects, budget stops, sandbox findings, and evaluation regressions. Tail-sample successful low-risk runs and high-volume read-only effects.

## Cross-run causality

Use a separate trace per workflow run, shared plan/experiment ID, span links, and artifact/decision lineage. Avoid week-long monolithic traces.

## Observability tests

Prove root spans, effect/journal correlation, invocation-attempt lineage, usage accuracy, absence of sensitive fields, failed-trace retention, cross-run links, bounded metric labels, and trace continuity across worker loss.

# Ports and adapters

> **Status: Informative guide.** The canonical registry and signatures live in [Reference interfaces](/reference/interfaces).

## Trusted context

Protected operations carry tenant, actor, run, correlation, deadline, classification, residency, and capability context. Effect invocations additionally carry effect and invocation identity.

## Port categories

| Category | Ports |
|---|---|
| Execution | `StateStorePort`, `RunJournalPort`, `DurableExecutionPort`, `WorkerLeasePort`, `CheckpointStorePort`, `EffectLedgerPort` |
| Capability | `ModelPort`, `ToolPort`, `MemoryPort`, `ArtifactStorePort`, `SandboxPort`, `HumanApprovalPort` |
| Governance | `PolicyPort`, `IdentityPort`, `SecretStorePort`, `EntitlementPort` |
| Evidence/integration | `EvaluationPort`, `TelemetryPort`, `AuditPort`, `MeteringPort`, `EventBusPort` |

Each port states responsibility, typed inputs/outputs, error taxonomy, sync/async and streaming semantics, idempotency, security boundary, and contract tests.

## Interoperability

MCP belongs at a tool/resource edge; it does not replace platform authorization, tenancy, durability, or audit.

A2A belongs at an external/federated agent edge. Remote messages and results remain untrusted and use explicit identity, capability, integrity, and schema contracts.

## Adapter constraints

- Temporal introduces deterministic workflow and activity-execution semantics.
- LangGraph introduces graph state, checkpoint, and interrupt semantics.
- Open Workflow Specification introduces a vendor-neutral DSL and subflow constructs.
- Dify and Langflow introduce native visual/component models when used as runtimes.
- Policy languages differ in conflict resolution and explanations.
- Sandboxes differ materially in startup latency and isolation.

Ports expose required semantics and typed extensions; they do not pretend every adapter is identical.

# Reference interfaces

> **Status: Normative reference.** TypeScript-like pseudocode expresses semantics, not a required implementation language. Machine-readable representations live under `/contracts`.

## Execution Kernel

```typescript
interface ExecutionKernel {
  evolve(state: RunState, event: RunEvent): RunState;
  decide(
    state: RunState,
    workflow: WorkflowVersion,
    deployment: DeploymentSnapshot
  ): readonly ExecutionIntent[];
}

type ExecutionIntent =
  | { type: "start_activity"; activityId: ActivityId; input: ActivityInputSnapshotRef }
  | { type: "restart_activity"; activityRunId: ActivityRunId; at: Instant }
  | { type: "request_effect"; effect: EffectDraft }
  | { type: "complete"; result: ExecutionResultDraft }
  | { type: "fail"; failure: RunFailure };
```

`ExecutionIntent` is a pure decision. Runtime-control writes and provider calls occur outside the kernel.

All semantically material waits and delegated starts use `request_effect`:

```text
human approval request       -> human.approval Effect
durable timer                -> timer.schedule Effect
external-event subscription  -> event.wait Effect
child WorkflowRun/AgentRun    -> child_run.start Effect
external-agent collaboration -> external_agent.task Effect
```

After the effect is durably planned, the kernel evolves the run into the corresponding waiting projection. Journal writes, lease changes, checkpoints, deterministic policy evaluation, and best-effort telemetry remain `RuntimeControlOperation`s and do not create recursive effects.

## Atomic execution commit

```typescript
interface LedgerMutation {
  entityId: string;
  operation:
    | "insert"
    | "transition"
    | "append_evidence"
    | "reconcile"
    | "reserve"
    | "release";
  expectedStatus?: string;
  nextStatus?: string;
  payloadDigest?: ContentDigest;
  payloadRef?: ProtectedPayloadRef;
}

interface TransitionCommit {
  events: readonly NewRunEvent[];
  nextState: RunState;
  effectMutations: readonly LedgerMutation[];
  invocationMutations: readonly LedgerMutation[];
  approvalMutations: readonly LedgerMutation[];
  budgetMutations: readonly LedgerMutation[];
  outboxEvents: readonly IntegrationEvent[];
}

interface ExecutionCommitPort {
  commit(
    ctx: ExecutionCommitContext,
    transition: TransitionCommit
  ): Promise<Result<CommitReceipt, CommitError>>;
}
```

The implementation **MUST** verify expected state version, run sequence, and current fencing token before committing the complete unit atomically. The serialized owner is `contracts/schemas/transition-commit.schema.json`.

`OpaqueCredentialHandle` is non-serializable, non-loggable, short-lived, audience-bound, and usable only by the dispatch adapter. The Execution Kernel, journal, run state, prompt compiler, and application layer never receive plaintext credentials.

## Capability attenuation

A child capability grant is calculated as an intersection of parent-delegable authority, task requirements, tenant entitlements, policy, data/region constraints, and budget allocation. An adapter or child **MUST NOT** broaden that result.

## Port registry

| Category | Ports |
|---|---|
| Inbound application | Agent/workflow/deployment/run/approval/evaluation/marketplace use cases |
| Execution | Kernel, atomic commit, state, journal, durable execution, worker lease, checkpoint, effect ledger |
| Capability | Model, tool, memory, artifact, sandbox, human approval, external agent |
| Governance | Policy, identity, entitlements, credential broker |
| Evidence/integration | Evaluation, telemetry, audit, metering, event bus |

## Contract tests

Adapters and infrastructure implementations prove:

```text
error classification and uncertain outcome
stream completion and cancellation
semantic effect identity across retries/attempts
fencing and stale-write rejection
atomic transition commit
idempotency and reconciliation
tenant/capability/data-region enforcement
credential non-disclosure
usage, audit, telemetry, and lineage correlation
capability negotiation and version drift
duplicate delivery and recovery
```

The shared conformance kit validates machine-readable contracts and representative failure cases.

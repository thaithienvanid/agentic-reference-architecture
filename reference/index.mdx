## Authority rule

Pages marked **Normative reference** define required names, representations, or interfaces. Pages marked **Reference profile** provide an optional concrete profile and become binding only when an implementation declares that profile. A **Reference checklist** maps canonical requirements to assessment evidence without creating semantic requirements.

Reference material states exact names, fields, interfaces, statuses, constraints, and evidence expectations. Design rationale belongs in guides or ADRs; implementation procedure belongs in how-to guides; end-to-end behavior belongs in worked examples.

# Reference

## Authority rule

| Page class | Authority |
|---|---|
| **Normative reference** | Required exact name, interface, lifecycle, registry, or representation |
| **Reference profile** | Optional concrete interoperability/transport form; binding when declared |
| **Reference checklist** | Maps canonical requirements to evidence; creates no new semantics |
| **Example fixture** | Executable representation test for one schema version |

When a machine contract and prose disagree about serialized shape, the named machine contract controls that representation. When a contract appears to alter semantic meaning, the specification controls and the mismatch is an erratum.

Reference material is optimized for lookup. Rationale belongs in guides or ADRs, implementation procedures in how-to guides, and complete behavior in [Enterprise worked examples](/examples/index).

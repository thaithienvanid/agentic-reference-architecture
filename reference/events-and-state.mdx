## Representation rules

- Omitted means not supplied or not applicable. `null` is used only when a schema assigns it a distinct meaning.
- Content digests use `sha256:` followed by exactly 64 hexadecimal characters.
- Status values and legal transitions come from [Lifecycle contracts](/reference/lifecycles).
- Historical events and completed results are immutable.
- Execution scope uses `ActivityRunId`, plus branch or iteration identity when present.
- Sensitive payloads remain behind protected references and digests.

## Activity result

## Effect

An `Effect` records one logical external or nondeterministic operation. Its semantic identity is stable across provider retries and whole-activity restarts when the requested operation and authority remain unchanged.

```json

  "effectId": "effect_01K2B",
  "semanticEffectSlot": "create-order",
  "semanticEffectKey": "effect-key:create-order:8ea7",
  "rootRunId": "run_01K2A",
  "runId": "run_01K2A",
  "activityRunId": "activity_run_create_order_01",
  "activityAttemptId": "activity_attempt_create_order_02",
  "executionScopePath": "run_01K2A/create-order",
  "kind": "tool.invoke",
  "toolRef": "purchase-order.create@4.2.1",
  "impact": {
    "determinism": "externally_determined",
    "dataAccess": "write",
    "mutation": "irreversible",
    "externality": "external_service",
    "compensability": "mitigatable"
  },
  "inputDigest": "sha256:6666666666666666666666666666666666666666666666666666666666666666",
  "authorityScopeDigest": "sha256:7777777777777777777777777777777777777777777777777777777777777777",
  "idempotencyKey": "tenant_acme_eu/effect/effect-key:create-order:8ea7",
  "reconciliationPolicyRef": "purchase-order-reconcile@2.0.0",
  "compensationPolicyRef": "purchase-order-cancel@1.0.0",
  "capabilityGrantRef": "capability_grant_01K2G",

```

`activityAttemptId` is provenance. It is not part of logical effect identity. A mutating effect declares idempotency, reconciliation, or both.

## Atomic transition commit

An authoritative transition carries the complete next state, typed ledger mutations, canonical events, and integration outbox events in one fenced consistency unit.

```typescript
interface TransitionCommit {
  events: readonly NewRunEvent[];
  nextState: RunState;
  effectMutations: readonly LedgerMutation[];
  invocationMutations: readonly LedgerMutation[];
  approvalMutations: readonly LedgerMutation[];
  budgetMutations: readonly LedgerMutation[];
  outboxEvents: readonly IntegrationEvent[];
}
```

The executable fixture demonstrates reconciliation of an uncertain purchase-order mutation. The implementation verifies `expectedStateVersion`, `expectedRunSequence`, `workerLeaseId`, and `fencingToken`, then commits the journal, current-state projection, ledgers, and outbox atomically. A conflict rejects the complete unit.

## Event categories and authority

- Domain events carry business meaning.
- Execution events describe run, activity, effect, invocation, approval, child-link, and lease lifecycles.
- Integration events cross bounded contexts through the outbox.
- Audit events carry independently protected security and governance evidence.
- Telemetry signals support operations and may be sampled.

The Platform Run Journal is authoritative for portable ARA execution facts. A broker, trace backend, durable-engine history, projection table, or checkpoint does not replace it. See [Lifecycle contracts](/reference/lifecycles), [Durable backend profile](/reference/durable-backend-profile), and [Contract catalog](/reference/contract-catalog).

# Manifest profile

> **Status: Reference profile.** Manifests are declarative control-plane inputs. Production compilation resolves references to exact immutable versions, locks, and digests. Machine-readable schemas live under `/contracts/schemas`.

Money follows [Canonical conventions](/reference/conventions) and is serialized as decimal strings.

```yaml
apiVersion: ara.dev/v1
kind: AgentVersion
metadata:
  name: procurement-order-agent
  version: 5.2.0
  owner: team-procurement

  purpose: Create policy-compliant purchase-order drafts
  entryWorkflow: procurement-order@3.0.0
  instructions:
    - procurement-agent-base@4.2.0
  modelRequirements:
    structuredOutput: true
    toolCalling: true
    minimumContextTokens: 64000
    allowedRegions: [eu-west]
    routingProfile: eu-enterprise-models@3.0.0
  capabilities:
    tools:
      - contract.read@2.1.0
      - vendor.read@3.0.0
      - purchase-order.create@4.2.1
    childAgents:
      - procurement-research-agent@2.0.0
      - procurement-compliance-agent@2.3.0
    maximumChildDepth: 1
  context:
    recipe: procurement-context@4.0.0
    memoryReadScopes: [workspace, agent]
    memoryWriteScopes: [agent]
  policyBundle: procurement-production-policy@7.0.0
  budgets:
    maximumCost: { amount: "12.00", currency: USD }
    maximumActivities: 60
    maximumModelEffects: 18
    maximumToolEffects: 25
    maximumWallTime: PT30M

    - procurement-correctness@6.0.0

```

An independent `AgentRun` created from this version owns one root `WorkflowRun` of `entryWorkflow`.

## Workflow version

```yaml
apiVersion: ara.dev/v1
kind: WorkflowVersion
metadata:
  name: procurement-order
  version: 3.0.0
spec:
  inputSchema: ProcurementTask@2
  outputSchema: ProcurementResult@3
  stateSchema: ProcurementRunState@3
  activities:
    - id: load-context
      type: task
      outputSchema: ProcurementContextRefs@1
    - id: research
      type: agent
      agent: procurement-research-agent@2.0.0
      executionMode: child
    - id: validate
      type: task
      input:
        research:
          fromActivityRun:
            activityId: research
            selection: single_required
    - id: approve
      type: human
      policy: high-value-purchase-approval@5.0.0
    - id: create-order
      type: tool
      tool: purchase-order.create@4.2.1
      semanticEffectSlot: create-purchase-order
  dependencies:
    - [load-context, research]
    - [research, validate]
    - [validate, approve]
    - [approve, create-order]
  completion:
    requiredActivities: [load-context, research, validate, approve, create-order]
```

Bindings identify exact execution instances or declared join/selection policies; they do not read an arbitrary latest activity output.

## Package lock

A resolver creates an exact immutable lock:

```yaml
apiVersion: ara.marketplace/v1
kind: PackageLock
metadata:
  id: package_lock_01K2P
  root: com.example.procurement-suite@2.4.0
resolved:
  - package: com.example.procurement-suite
    version: 2.4.0
    digest: sha256:6a14...
  - package: com.example.document-tools
    version: 3.4.2
    digest: sha256:3d91...
resolver:
  version: ara-package-resolver@1.0.0
  generatedAt: 2026-07-13T10:00:00Z
lockDigest: sha256:package-lock...
```

An activated installation MUST use a `PackageLock`; unresolved ranges are prohibited.

Granted permissions MUST be a subset of requested permissions and tenant/platform authority.
